# 1. var、let和const区别

# 1 前言

随着2015年6月 ECMAScript 6.0 发布以来，有几个新的变量声明方式进入前端开发者的视线，即 let、const、import 和 class 命令，而这些新的声明方式之中有两个容易和 var 混淆，那就是 let 和 const。

# 2 简单介绍

let 的用法类似于 var，但是 let 只在所在的代码块内有效，所以我们一般使用 let 替代 var。而 const 用来声明常量。

声明方式 变量提升 暂时性死区 重复声明 初始值
var 允许 不存在 允许 不需要
let 不允许 存在 不允许 不需要
const 不允许 存在 不允许 需要

# 3 变量提升

//正常运行，控制台输出 未定义的的值
console.log(a);
var a = 1;

//报错，未捕获的ReferenceError: b没有定义
console.log(b);
let b = 1;

//报错，未捕获的ReferenceError: c没有定义
console.log(c);
const c = 1;

var 命令经常会发生变量提升现象，按照逻辑是先声明后使用。从而Es6规定let，const不发生变量提升，使用 let 和 const 命令声明变量之前，该变量是不可用的。

# 4 暂时性死区

概述：如果在代码块中存在 let 或 const 命令，这个区块对这些命令声明的变量，从一开始就形成了封闭作用域。凡是在声明之前就使用这些变量，就会报错。

var name = jack;

if (true) {
	name = '张三';
    //报错，未捕获的ReferenceError: name没有定义
	let name;
}

这段代码的原意是在 if 内定义外部的 name 为 '张三'。

但现实是，存在全局变量 name，但是块级作用域内 let 又声明了一个 name变量，导致后者被绑定在这个块级作用域中，所以在 let 声明变量前，对name 赋值就报错了。

# 5 重复声明

概述：指在相同作用域内，重复声明同一个变量。

let 和 const 命令声明的变量不允许重复声明：

function a (){
	let a = 10;
	const b =3;
	//报错
	let a = 11;
	const b = 3.14;
}

var 是允许重复定义的，而这又会给我们带来什么麻烦呢？

var i = 10;
for(var i = 0;i < 5;i++){
  console.log(i);
}
console.log(i);// 输出 5

因为var没有块级作用域，导致for循环的i覆盖外面的i,所以最后输出的i的值为5。

# 5.1 初始值

由于 const 声明的是只读的常量，一旦声明，就必须立即初始化，声明之后值不能改变。

# 5.2 作用域

由于 ES5 中只有全局作用域和函数作用域，没有块级作用域，这带来很多不合理的场景。

  • 第一种场景，内置变量被外部覆盖

    var text = new Date();//全局作用域

function a() {
  console.log(text);//局部作用域
  if (false) {
    var text = 'hello world';
  }
}

a(); // 未定义的

  • 第二种场景，用来计数的循环变量泄露为全局变量

    var i = 10;
for(var i = 0;i < 5;i++){
  console.log(i);
}
console.log(i);// 输出 5

    解析：变量i只用来控制循环，但是循环结束后，它并没有消失，泄露成了全局变量。
